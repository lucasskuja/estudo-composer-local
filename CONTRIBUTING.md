# Contributing

Contributions should strengthen this repository as a reusable reference asset for local Cloud Composer development.

## Contribution Principles

Changes should:

- improve clarity, reuse, or maintainability
- stay aligned with the actual repository scope
- avoid overstating capabilities that are not implemented here
- preserve the distinction between this repository and the upstream Google project

## Good Contributions

Examples of useful contributions include:

- improving documentation accuracy
- clarifying setup and reuse patterns
- adding lightweight repository conventions for extension work
- documenting safe ways to build on top of the upstream submodule

## Out Of Scope

The following changes should be discussed before being added:

- large custom frameworks presented as part of the current baseline
- documentation that implies production-ready capabilities not present in the repository
- modifications that blur ownership between this repository and `composer-local-dev`

## Working Model

When contributing:

1. Keep claims consistent with what the repository actually contains.
2. Prefer practical, reusable patterns over tutorial-style explanations.
3. Treat the upstream submodule as the runtime dependency and this repository as the reference wrapper.

## Pull Requests

Pull requests should include:

- a clear summary of the change
- the reason the change improves the repository as a reusable reference
- documentation updates when repository behavior or usage expectations change

## Questions

If a contribution would materially change the repository scope, architecture direction, or positioning, align on that change before implementation.
